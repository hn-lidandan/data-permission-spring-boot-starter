### 把 SDK 开发的核心规范精简为四个字：“省心、严谨”。具体可以总结为以下五条军规：

1、零克制（管好依赖）： 坚决不用大框架（如 Spring），尽量零依赖或极少依赖。对必须引入且易冲突的包进行 Shade（重命名遮蔽），绝不给接入方制造 Jar 包冲突。

2、不甩锅（防御编程）： 入参必判空，网络必设超时。内部异常绝不静默吞掉，统一转换为自定义的 SdkException 抛出，保证线程安全。

3、别念旧（向下兼容）： 严格遵守语义化版本（X.Y.Z）。升级时不盲目修改或删除旧 API，淘汰接口必须加 @Deprecated 并指明新出路。

4、少废话（开箱即用）： 提供 Builder 模式和合理的默认配置。使用 SLF4J 面向接口打日志，不乱刷接入方的控制台。

5、看注释（自解释性）： 所有公开的类和方法必须写满 Javadoc 注释，接口命名统一（XxxRequest/XxxResponse），让别人看代码提示就能直接上手。
