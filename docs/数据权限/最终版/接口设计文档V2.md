# 数据权限服务接口设计文档（最终版）

## 1. 接口补齐结论

现有初稿已覆盖“获取用户权限上下文”和“资源权限校验”，但作为可落地开发文档仍缺少以下接口：

| 分类 | 需补齐接口 | 说明 |
| --- | --- | --- |
| SDK 执行面 | 生成查询过滤条件 | 业务服务需要拿到参数化过滤条件，由 SDK 注入 PostgreSQL / Doris SQL AST。 |
| 管理面 | 策略启停、策略模拟 | 只做新增/修改/删除不足以支持真实运营，策略上线前需要模拟验证。 |
| 管理面 | 上下文刷新 | 策略、角色、部门、用户组、资产组、日志源变更后必须主动刷新或失效缓存。 |
| 管理面 | 审计详情查询 | 审计列表只能筛选，排查越权问题时需要查看单条完整上下文。 |

接口设计统一采用“管理面集中配置、SDK 本地执行”的原则：权限服务返回参数化规则与权限上下文，SDK 在业务服务内完成 SQL 改写、字段处理、导出拦截和审计上报。

## 2. 通用规范

### 2.1 基础路径

| 接口类型 | Base Path | 调用方 |
| --- | --- | --- |
| SDK 执行接口 | `/api/v1/permission` | Java SDK、Rust SDK、业务服务 |
| 管理端接口 | `/api/v1/permission` | 权限管理后台、系统管理服务 |

### 2.2 请求头

| Header | 必填 | 说明 | 示例 |
| --- | --- | --- | --- |
| `Authorization` | 是 | 调用方访问令牌，管理端接口必须校验管理员权限。 | `Bearer eyJhbGci...` |
| `Content-Type` | 是 | 固定为 JSON。 | `application/json` |
| `X-Trace-Id` | 否 | 链路追踪 ID；为空时服务端生成。 | `trc-20260629-0001` |
| `X-Client-App` | 否 | 调用方应用标识。 | `risk-center` |
| `X-Operator-Id` | 管理端必填 | 管理端操作人用户 ID，用于审计。 | `admin_001` |

说明：`tenant_id`、`user_id` 等权限判断字段必须以服务端可信身份为准。业务请求体中的身份字段仅用于 SDK 与权限服务之间的内部协议，不能由前端直接传入后覆盖服务端上下文。

### 2.3 通用响应结构

```json
{
  "code": "0",
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | string | 是 | 业务响应码，`0` 表示成功。 |
| `message` | string | 是 | 响应说明。 |
| `trace_id` | string | 是 | 链路追踪 ID。 |
| `data` | object / array / null | 是 | 业务数据。 |

### 2.4 通用分页结构

```json
{
  "code": "0",
  "message": "success",
  "data": {
    "offset": 1,
    "limit": 20,
    "total": 128,
    "items": []
  }
}
```

### 2.5 通用枚举

| 枚举 | 值 | 说明 |
| --- | --- | --- |
| `resource_type` | `doris`、`postgre_sql` | 数据引擎或资源归属类型，和表设计保持一致。 |
| `action` | `QUERY`、`DETAIL`、`EXPORT`、`DELETE`、`MANAGE` | 操作动作。 |
| `scene` | `QUERY`、`DETAIL`、`EXPORT` | 字段策略适用场景。 |
| `dept_scope_type` | `ALL`、`CURRENT`、`SUB`、`SELF`、`CUSTOM` | 部门数据范围。 |
| `permission_type` | `show`、`mask`、`hide`、`deny` | 字段权限类型。 |
| `mask_rule_type` | `PHONE`、`EMAIL`、`ID_CARD`、`IP`、`TOKEN`、`REGEX` | 脱敏规则类型。 |
| `effect` | `allow`、`deny` | 策略效果，`deny` 优先。 |
| `audit_result` | `ALLOWED`、`DENIED`、`SUCCESS`、`FAILED` | 审计结果。 |

## 3. SDK 执行接口

### 3.1 获取用户权限上下文

用于 SDK 登录后、请求进入业务服务后或缓存过期时获取用户可访问范围。权限服务负责合并用户、角色、部门、用户组授权并返回打平后的上下文。

```http
POST /api/v1/permission/context/query
Content-Type: application/json
```

入参说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `tenant_id` | string | 是 | 租户 ID。 |
| `user_id` | string | 是 | 用户 ID。 |
| `dept_id` | string | 是 | 用户所属部门 |
| `role_id` | string | 是 | 用户角色 |

请求示例：

```json

{
  "tenant_id": "t001",
  "user_id": "u001",
  "dept_id":"xxxx",
  "role_id":"xxxxxx"

}

```

出参说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `user_id` | string | 用户 ID。 |
| `tenant_id` | string | 租户 ID。 |
| `dept_scope` | array | 可访问部门 ID 列表，`["*"]` 表示全部。 |
| `asset_group_scope` | array | 可访问资产组 ID 列表，`["*"]` 表示全部。 |
| `data_level_scope` | array | 可访问数据等级列表。 |
| `resource_object` | object | 可访问资源对象，按 `doris`、`postgre_sql` 分组。 |
| `field_policies` | object | 字段策略列表。 |
| `export_policies` | object | 导出策略列表。 |
| `deny_all` | boolean | 是否默认拒绝。 |
| `version` | integer | 权限上下文版本号。 |
| `expires_at` | string | 上下文过期时间，ISO 8601。 |

响应示例：

```json

{

    "user_id": "u001",
    "tenant_id": "t001",
    "dept_scope": ["d001", "d002"],
    "asset_group_scope": ["ag001", "ag002"],
    "data_level_scope": ["public", "internal"],
    "resource_object": {
      "doris": ["security_log", "alert_event"],
      "postgre_sql": ["asset", "report"]
    },
    "field_policies": [
      {
        "field_name": "phone",
        "permission_type": "MASK",
        "mask_regex": "PHONE",
        "scene": ["DETAIL"]
      },
      {
        "field_name": "raw_log",
        "permission_type": "HIDE",
        "mask_regex": null,
        "scene": ["DETAIL","EXPORT"]
      }
    ],
    "export_policies": [
      {
        "resource_type": "doris",
        "resource_id": "security_log",
        "is_allowed": true,
        "max_rows": 5000,
        "need_approval": false
      }
    ],
    "deny_all": false,
    "version": 12,
    "expires_at": "2026-06-29T18:00:00+08:00"
}


```



## 4. 管理端接口

管理端接口必须校验管理员身份，并记录 `POLICY_CHANGE` 或 `CONTEXT_REFRESH` 审计。新增、修改、删除、启停策略后，需要发布权限上下文失效事件。

### 4.1 数据权限策略管理

#### 4.1.1 新增数据权限策略

```http

Content-Type: application/json
```

入参说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `policy_name` | string | 是 | 策略名称。 |

| `tenant_id` | string | 是 | 所属租户。 |
| `target_type` | string | 是 | 授权目标类型 USER/ROLE/DEPT |
| `target_dept_id` | List<string> | 否 | 授权目标部门 ID。 |
| `target_role_id` | List<string> | 否 | 授权目标角色 ID。 |
| `target_user_id` | List<string> | 否 | 授权目标用户 ID。 |
| `target_user_group_id` | List<string> | 否 | 授权目标用户组 ID。 |
| `dept_scope_type` | string | 是 | 部门范围类型。 |
| `custom_dept_ids` | array | 否 | 自定义部门范围，`dept_scope_type=CUSTOM` 时必填。 |
| `data_levels` | array | 是 | 数据等级，如 ["public","internal","sensitive"]|
| `asset_group_scope` | array | 否 | 资产组范围，`["*"]` 表示全部。 |
| `resource_object` | object | 是 | 可访问资源对象。 |
| `field_policy_ids` | array | 否 | 关联字段策略 ID。 |
| `export_policy_ids` | array | 否 | 关联导出策略 ID。 |
| `effect` | string | 是 | `allow` 或 `deny`。 |
| `priority` | integer | 是 | 优先级，数字越大优先级越高。 |
| `is_enabled` | boolean | 是 | 是否启用。 |

请求示例：

```json

{
{

  "target_dept_ids": null,

  "target_user_ids": null,
  "target_user_group_ids": null,
  "dept_scope_type": "SUB",
  "custom_dept_ids": [],
  "data_level_scope": ["internal"],
  "asset_group_scope": ["ag001", "ag002"],
  "resource_object": {
    "doris": ["security_log", "alert_event"],
    "postgre_sql": ["asset"]
  },
  "field_policy_ids": ["fp_1001", "fp_1002"],
  "export_policy_ids": ["er_2001"],
  "effect": "allow",
  "priority": 100,
  "is_enabled": true
}

```

响应示例：

```json

{

    "id": "101",
    "is_success": false,
    "error_message": "XX异常"
}

```

出参说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | 新增策略 ID。 |
| `is_success` | boolean | 是否标识 |
| `error_message` | string | 失败原因 |

## 6. 错误码

| code | HTTP 状态码 | 说明 | SDK 建议处理 |
| --- | --- | --- | --- |
| `0` | 200 | 成功。 | 正常处理。 |
| `DP_400_PARAM_INVALID` | 400 | 请求参数错误。 | 拒绝本次访问并记录审计。 |
| `DP_401_UNAUTHORIZED` | 401 | 未认证或令牌失效。 | 拒绝本次访问。 |
| `DP_403_FORBIDDEN` | 403 | 调用方无接口权限。 | 拒绝本次访问。 |
| `DP_404_NOT_FOUND` | 404 | 资源不存在。 | 管理端提示不存在；SDK 按无权限处理。 |
| `DP_409_VERSION_CONFLICT` | 409 | 上下文版本冲突。 | SDK 重新拉取上下文。 |
| `DP_422_POLICY_INVALID` | 422 | 策略配置不合法。 | 管理端提示修正配置。 |
| `DP_429_TOO_MANY_REQUESTS` | 429 | 调用过于频繁。 | SDK 使用本地缓存；无缓存时默认拒绝。 |
| `DP_500_INTERNAL_ERROR` | 500 | 服务内部错误。 | 默认拒绝或返回空结果。 |
| `DP_503_CONTEXT_UNAVAILABLE` | 503 | 权限上下文不可用。 | 默认拒绝或将 SQL 改写为 `1 = 0`。 |

错误响应示例：

```json
{
  "code": "DP_503_CONTEXT_UNAVAILABLE",
  "message": "permission context is unavailable",
  "trace_id": "trc-20260629-9999",
  "data": {
    "deny_all": true,
    "reason": "context_compute_failed"
  }
}
```
